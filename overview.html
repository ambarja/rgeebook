Overview

What is Google Earth Engine?

Google Earth Engine (GEE) is a cloud-based platform that helps access high-performance computing resources for processing and analyzing large geospatial datasets (Golerick et al.,2017). At the moment, GEE only supports client libraries for the JavaScript and Python programming languages. Check out its official GitHub repository for more details: https://github.com/google/earthengine-api.

client library: Sometimes known as a helper library, is a collection of code that help to communicate more effectively with an API. For example, if you want to invoke certain image asset in GEE, you must type the following:

# Connecting to GEE server via POST request.
{
   "functionInvocationValue": {
      "functionName": "Image.load",
      "arguments": {
         "id": {
            "constantValue": "USGS/SRTMGL1_003"
         }
      }
   }
}

However, using the GEE client library, the code is reduced to only one line of code!

# Connecting to GEE server via client library + web API.
ee$Image("USGS/SRTMGL1_003")

What is rgee?

rgee is the simplest way for using GEE from within R. Using reticulate in the back-end, rgee connects R and Python. Therefore, you may think on rgee as a nested wrapper. One wrapper to connect R to the Python Earth Engine API, while the other connects Python to the Web REST API (see figure below).

When an Earth Engine request is created in R, reticulate translate the R code into Python and send it to the Earth Engine Python client library, which converts Python code in JSON format. Then, the GEE Platform receives the JSON request through a Web REST API. The response takes the same path as before, but in reverse.

rgee Guides

The guides are written for R with Rstudio. The R Quickstart guide and the tutorials describe basic concepts and example workflows to help beginning users. The guides have several examples for both rgee and rgeeExtra.

The Best Practices and Debugging guides have more complex examples and provide techniques useful for overcoming errors and scaling your Earth Engine analyses to larger areas, longer time series and more data. More advanced Earth Engine concepts provide necessary background for understanding how the Earth Engine service works and writing effective code.

The remainder of the guides are intended to illustrate important concepts about data types such as:

  • Image, The fundamental raster data type in Earth Engine.
  • ImageCollection, a stack or time-series of images.
  • Geometry, the fundamental vector data type in Earth Engine.
  • Feature, or a Geometry with attributes.
  • FeatureCollection, or a set of features.
  • Reducer, an object used to compute statistics or perform aggregations.
  • Join, or how to combine datasets (Image or Feature collections) based on time, location, or an attribute property.
  • Array, for multi-dimensional analyses.

There are also sections for machine learning, specialized or sensor specific algorithms (e.g. Landsat algorithms), and Rstudio addins, shinyapps.io apps, and data (asset) management.
